# gstack-plan：全面审阅关卡

当用户想要规划 Claude Code 项目时由协调器注入。追加到现有的 CLAUDE.md 中。

## 规划流水线
1. 读取 CLAUDE.md 并理解项目上下文。
2. 运行 /office-hours 以产出设计文档（问题陈述、前提、备选方案）。
3. 运行 /autoplan 以审阅设计（CEO + 工程 + 设计 + DX 审阅 + Codex 对抗性审阅）。
4. 将最终审阅过的计划保存到协调器可以后续引用的文件中。
   写入路径：plans/<project-slug>-plan-<date>.md（在当前仓库中）。
   包含设计文档、所有审阅决策和实现序列。
5. 向协调器汇报：
   - 计划文件路径
   - 所设计内容及其关键决策的单段摘要
   - 已接受的范围扩展列表（如有）
   - 推荐的下一步（通常是：使用 gstack-full 生成一个新会话以执行）

不要执行任何操作。此流程仅供规划。
协调器会将计划链接持久化到其自己的内存/知识存储中。
