# QA 问题分类体系

## 严重程度级别

| 严重程度 | 定义 | 示例 |
|----------|------|------|
| **critical（严重）** | 阻断核心工作流、导致数据丢失或应用崩溃 | 表单提交导致报错页面、结账流程数据丢失、数据未经确认即被删除 |
| **high（高）** | 主要功能损坏或无法使用，无可用变通方案 | 搜索结果返回错误、文件上传静默失败、身份验证重定向循环 |
| **medium（中）** | 功能可用但存在明显问题，存在变通方案 | 页面加载缓慢（>5秒）、表单缺少验证但提交仍可正常工作、仅在移动端布局错乱 |
| **low（低）** | 轻微的外观或优化问题 | 页脚拼写错误、1像素对齐问题、悬停状态不一致 |

## 问题分类

### 1. 视觉/UI
- 布局错乱（元素重叠、文本被截断、出现横向滚动条）
- 图片损坏或缺失
- z-index 不正确（元素显示在其他元素后面）
- 字体/颜色不一致
- 动画异常（卡顿、过渡效果不完整）
- 对齐问题（未对齐到网格、间距不均匀）
- 深色模式/主题问题

### 2. 功能性
- 链接失效（404、目标地址错误）
- 按钮无响应（点击无任何效果）
- 表单验证（缺失、错误、可被绕过）
- 重定向不正确
- 状态未持久化（刷新后数据丢失、后退按钮问题）
- 竞态条件（重复提交、数据过期）
- 搜索返回错误结果或无结果

### 3. 用户体验（UX）
- 导航令人困惑（无面包屑导航、死路）
- 缺少加载指示器（用户不知道系统正在处理）
- 交互响应慢（>500毫秒且无任何反馈）
- 错误信息不清晰（"出了点问题"但无具体细节）
- 破坏性操作前无确认提示
- 不同页面的交互模式不一致
- 死路（无法返回、无下一步操作建议）

### 4. 内容
- 拼写和语法错误
- 文本过时或不准确
- 占位符/lorem ipsum 文本未删除
- 文本被截断（超出部分未显示省略号或"更多"）
- 按钮或表单字段标签错误
- 缺少或无帮助的空状态提示

### 5. 性能
- 页面加载缓慢（>3秒）
- 滚动卡顿（掉帧）
- 布局偏移（内容加载后跳动）
- 过多的网络请求（单页>50个）
- 未优化的大型图片
- 阻塞型 JavaScript（加载期间页面无响应）

### 6. 控制台/错误
- JavaScript 异常（未捕获的错误）
- 网络请求失败（4xx、5xx）
- 弃用警告（即将出现的破坏性变更）
- CORS 错误
- 混合内容警告（HTTPS 页面中嵌入 HTTP 资源）
- CSP（内容安全策略）违规

### 7. 无障碍性
- 图片缺少 alt 文本
- 表单输入框未加标签
- 键盘导航损坏（无法通过 Tab 键移动到元素）
- 焦点陷阱（无法离开模态框或下拉菜单）
- 缺少或错误的 ARIA 属性
- 颜色对比度不足
- 屏幕阅读器无法获取内容

## 逐页面探索清单

在 QA 会话中访问每个页面时：

1. **视觉扫描** — 截取带标注的截图（`snapshot -i -a -o`）。检查布局问题、图片损坏、对齐问题。
2. **交互元素** — 点击每个按钮、链接和控件。检查每一项是否按其标注工作。
3. **表单** — 填写并提交。测试空提交、无效数据、边缘情况（超长文本、特殊字符）。
4. **导航** — 检查所有进/出路径。面包屑导航、返回按钮、深度链接、移动端菜单。
5. **状态** — 检查空状态、加载状态、错误状态、满数据/溢出状态。
6. **控制台** — 交互后运行 `console --errors`。是否有新的 JS 错误或失败的请求？
7. **响应式** — 如有需要，检查移动端和平板端视口。
8. **认证边界** — 注销时会发生什么？不同用户角色有何不同？
