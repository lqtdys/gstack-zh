# TODOS.md 格式参考

规范 TODOS.md 格式的共同参考。由 `/ship`（步骤 5.5）和 `/plan-ceo-review`（TODOS.md 更新部分）引用以确保一致的 TODO 项目结构。

---

## 文件结构

```markdown
# TODO

## <技能/组件>     ← 例如：## 浏览，## Ship，## 评审，## 基础设施
<项目按 P0 在前、其次 P1/P2/P3/P4 排序>

## 已完成
<已完成项目附带完成注释>
```

**章节：** 按技能或组件分组（`## 浏览`、`## Ship`、`## 评审`、`## QA`、`## 回顾`、`## 基础设施`）。各章节内按优先级排序（P0 在顶部）。

---

## TODO 项目格式

每项是各章节下的 H3：

```markdown
### <标题>

**内容：** 工作的一行描述。

**原因：** 解决的具体问题或释放的价值。

**背景：** 足够的细节，使得任何三个月后开始做这件事的人都能理解动机、当前状态和开始地点。

**工作量：** S / M / L / XL
**优先级：** P0 / P1 / P2 / P3 / P4
**前置：** <先决条件，或 "无">
```

**必填字段：** 内容、原因、背景、工作量、优先级
**可选字段：** 前置、阻塞于

---

## 优先级定义

- **P0** — 阻塞：必须在下个版本之前完成
- **P1** — 关键：应在当前迭代中完成
- **P2** — 重要：在 P0/P1 完成后处理
- **P3** — 锦上添花：采用率/使用数据出来后重新审视
- **P4** — 将来：好的想法，没有急迫性

---

## 已完成项目格式

项目完成后，移到 `## 已完成` 章节，保留其原始内容并附加：

```markdown
**完成：** vX.Y.Z (YYYY-MM-DD)
```
